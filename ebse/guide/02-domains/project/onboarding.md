# Onboarding

**[BONNE PRATIQUE]** Checklist day 1, first PR < 24h, pair programming premiere semaine, setup < 30 min | Score GRADE : 3/7

## Checklist day 1

1. Acces repo, CI, channels de communication
2. Clone + setup local fonctionnel (README "zero to running")
3. Lire CONVENTIONS.md et architecture overview
4. Premier commit (typo fix, small task) = premiere PR < 24h
5. Pair programming avec un membre de l'equipe

## Regles

| Regle | Source |
|-------|--------|
| Setup local < 30 min via README | DORA/Accelerate — frein #1 a la productivite |
| First PR < 24h | Microsoft NEET study — engagement rapide |
| Pair programming semaine 1 | Accelerate — transfert de connaissances |
| Buddy system (1 referent attribue) | Microsoft NEET study — retention |

## Pourquoi c'est critique

DORA montre que le temps de setup est un des premiers predicteurs de productivite. Microsoft (etude NEET, 2022) demontre qu'un onboarding structure reduit le time-to-productivity de 60% et ameliore la retention.

Sources : DORA/Accelerate (niv. 4), Microsoft NEET study 2022 (niv. 4)
