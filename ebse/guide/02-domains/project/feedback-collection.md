# Feedback Collection

**[CHOIX D'EQUIPE]** Templates structures pour bug reports, triage hebdomadaire, GitHub Issues + labels | Score GRADE : 2/7

## Template bug report

```markdown
**Description** : [Quoi]
**Etapes de reproduction** : 1. ... 2. ... 3. ...
**Resultat attendu** : [Ce qui devrait arriver]
**Resultat actuel** : [Ce qui arrive]
**Environnement** : [OS, navigateur, version]
**Captures** : [Screenshots si pertinent]
```

## Processus

| Etape | Detail |
|-------|--------|
| Collecte | Template oblige via GitHub Issue forms |
| Triage | Hebdomadaire — prioriser par impact utilisateur |
| Labels | `bug`, `ux`, `feature-request`, `low/medium/high` |
| Feedback loop | Informer le rapporteur quand corrige |

## Point cle

Un bug report sans etapes de reproduction est inutile. Le template force la structure et reduit les allers-retours. L'utilisateur doit comprendre que son retour a ete pris en compte (feedback loop).
